# Demo Cases

本文提供 MVP 阶段的 3 个演示案例，分别覆盖 STM32、ESP32、NXP RT1064。每个案例都用于验证嵌入式 AI 助手能否输出硬件方案、淘宝搜索链接、接线说明、代码骨架和调试步骤。

## Case 1：STM32 蓝桥杯单片机训练题

### 场景

蓝桥杯单片机组训练。用户希望实现温度采集、按键切换和数码管显示。

### 用户输入

```text
我想做一个蓝桥杯单片机组训练题：用温度传感器采集温度，通过按键切换显示模式，并用数码管显示结果。请直接给我可以落地的方案和 STM32 标准库代码骨架。
```

### 期望识别

- `mode`: `delivery`
- `scenario`: `lanqiao_mcu`
- `mcu_family`: `STM32`
- `code_style`: `standard_peripheral_library`

### 期望硬件

| 硬件 | 淘宝搜索关键词 | 用途 |
| --- | --- | --- |
| STM32F103C8T6 最小系统板 | `STM32F103C8T6 最小系统板` | 主控训练 |
| DS18B20 温度传感器 | `DS18B20 温度传感器模块` | 温度采集 |
| 4 位数码管模块 | `四位数码管模块 共阴` | 显示结果 |
| 独立按键模块 | `单片机 按键模块` | 模式切换 |
| 杜邦线 | `杜邦线 公对母` | 接线 |

### 接线示例

| 模块 | 模块引脚 | STM32 引脚 | 说明 |
| --- | --- | --- | --- |
| DS18B20 | VCC | 3.3V | 供电 |
| DS18B20 | GND | GND | 共地 |
| DS18B20 | DQ | PB12 | 单总线数据 |
| 按键 | OUT | PA0 | 模式切换 |
| 数码管 | SEG/PIN | GPIOB/GPIOA | 按实际模块调整 |

### 代码骨架要求

- `main.c`：初始化与主循环
- `bsp_key.c`：按键扫描
- `bsp_ds18b20.c`：温度读取
- `bsp_seg.c`：数码管动态扫描
- `bsp_timer.c`：定时器中断

### 调试步骤

1. 先点亮 LED，确认 GPIO 和下载流程正常。
2. 单独调试按键，确认消抖和模式切换。
3. 单独调试数码管动态扫描，确认显示稳定。
4. 单独读取 DS18B20 温度，确认时序正确。
5. 最后把按键、温度、显示合并。

### 验收标准

- AI 输出淘宝搜索链接。
- AI 输出 STM32 标准库风格代码。
- AI 明确指出 DS18B20 需要上拉电阻。
- AI 给出分阶段调试步骤。

## Case 2：ESP32 物联网传感器节点

### 场景

电子设计竞赛或课程设计。用户希望用 ESP32 采集温湿度，并通过 Wi-Fi 输出数据。

### 用户输入

```text
教我用 ESP32 做一个温湿度采集节点，可以通过 Wi-Fi 把数据发到网页或串口显示。我是初学者，希望你一步一步教我。
```

### 期望识别

- `mode`: `teaching`
- `scenario`: `electronic_design`
- `mcu_family`: `ESP32`
- `code_style`: `arduino_for_esp32`

### 期望硬件

| 硬件 | 淘宝搜索关键词 | 用途 |
| --- | --- | --- |
| ESP32 DevKit 开发板 | `ESP32 DevKit 开发板` | 主控和 Wi-Fi |
| DHT11 或 DHT22 | `DHT22 温湿度传感器模块` | 温湿度采集 |
| 面包板 | `面包板 830孔` | 快速搭建 |
| 杜邦线 | `杜邦线 公对母` | 接线 |

### 接线示例

| 模块 | 模块引脚 | ESP32 引脚 | 说明 |
| --- | --- | --- | --- |
| DHT22 | VCC | 3.3V | 供电 |
| DHT22 | GND | GND | 共地 |
| DHT22 | DATA | GPIO4 | 数据引脚 |

### 代码骨架要求

- 使用 Arduino for ESP32。
- `setup()` 初始化串口、Wi-Fi 和传感器。
- `loop()` 定时读取温湿度。
- 可先串口输出，再扩展 Web Server。

### 调试步骤

1. 先用 Blink 确认开发板和环境正常。
2. 打开串口监视器，确认波特率。
3. 单独读取 DHT22。
4. 再连接 Wi-Fi。
5. 最后做网页显示或 HTTP 接口。

### 验收标准

- AI 用初学者能理解的方式解释。
- AI 输出 Arduino for ESP32 示例。
- AI 给出库安装说明。
- AI 不把 ESP-IDF 当第一版默认方案。

## Case 3：NXP RT1064 智能车控制训练

### 场景

智能车训练。用户希望使用 NXP RT1064 做电机 PWM 控制、编码器测速和 PID 调速。

### 用户输入

```text
我想用 NXP RT1064 做智能车训练，先实现两个直流电机的 PWM 控制和编码器测速，再做简单 PID 调速。直接给落地方案、接线和 MCUXpresso SDK 代码思路。
```

### 期望识别

- `mode`: `delivery`
- `scenario`: `smart_car`
- `mcu_family`: `NXP_RT1064`
- `code_style`: `mcuxpresso_sdk`

### 期望硬件

| 硬件 | 淘宝搜索关键词 | 用途 |
| --- | --- | --- |
| NXP RT1064 开发板 | `NXP RT1064 开发板` | 主控 |
| TB6612FNG 电机驱动 | `TB6612FNG 电机驱动模块` | 电机驱动 |
| 带编码器直流减速电机 | `带编码器 直流减速电机` | 执行器与测速 |
| 电池和降压模块 | `智能车 电池 降压模块` | 供电 |
| 杜邦线 | `杜邦线 公对母` | 接线 |

### 接线示例

| 模块 | 模块引脚 | RT1064 引脚 | 说明 |
| --- | --- | --- | --- |
| TB6612FNG | PWMA | PWM 引脚 | 左电机速度 |
| TB6612FNG | AIN1/AIN2 | GPIO | 左电机方向 |
| TB6612FNG | PWMB | PWM 引脚 | 右电机速度 |
| 编码器 | A/B | 定时器/捕获引脚 | 测速 |
| 电机电源 | VM | 电池输出 | 与逻辑电源区分 |

### 代码骨架要求

- 使用 MCUXpresso SDK。
- 说明 `board.c`、`pin_mux.c`、`clock_config.c` 依赖。
- 输出 PWM 初始化、GPIO 方向控制、编码器计数、PID 更新伪代码。
- 不伪造完整工程。

### 调试步骤

1. 单独输出 PWM，用示波器或万用表确认占空比变化。
2. 接电机驱动但不装车，空载测试方向。
3. 读取编码器计数，确认正反转符号。
4. 加入速度闭环，先低速测试。
5. 最后上车联调 PID。

### 验收标准

- AI 明确区分电机电源和逻辑电源。
- AI 输出 MCUXpresso SDK 风格代码思路。
- AI 提醒共地和电机反灌干扰。
- AI 给出从 PWM 到 PID 的分阶段调试流程。
